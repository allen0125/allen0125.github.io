创建 3 个 Linode 节点并完成 Hadoop 的搭建

还是没能忍住，想在开始学之前就给自己搭建一套能用的 Hadoop 集群，倒是也好，先搭建一套出来跑通了， 然后在学习的过程中会对搭建的每一步都有更加深刻的理解和认识，所以我参考 Linode 官方示例完成了 三个节点的 Hadoop 集群搭建。

首先需要创建 3 个 Linode 节点，节点名分别为：node-master，node1，node2，创建时需要给每一个 节点都添加 Private IP，也就是内网 IP。node-master 的内存不要小于 2Gb，我最初选的是 1C1G 的配置后来改成了 2C4G 的配置。其它两个节点全都使用了 1C1G 的配置。

配置 /etc/hosts文件的时候，例如下面这个内容需要配置在 IPv4 的位置，不能写在文件最后，文件最后 应该是属于 IPv6 的位置。

192.168.134.111     node-master
192.168.138.222     node1
192.168.193.333     node2

剩下就没什么需要注意的地方了，跟着文章最后的参考链接做然后把服务跑起来就好了。

深入理解 Hadoop 第一章读书笔记

所有大数据计算处理模式都有一个目的，就是使输入/输出（I/O）并行化，从而提高数据处理性能

大数据：1、把数据分发到多个节点；2、把计算逻辑移动到数据附近；3、计算节点进行本地数据处；4、 有限顺序读，次之随机读；

大数据编程模型：大规模并行处理系统 MMP 系统、内存数据库系统、MapReduce 系统、整体同步并行系统 （BSP）。机器学习相关的并行计算应该就是 BSP 系统的实现。

Amdhal 定律：揭示了通过往集群增加更多计算节点的方法来提高集群性能的极限值。

FAQ:

1: 为什么要使用 Linode 呢？

答: 因为博主一直都在使用 Linode 的服务，使用 Linode 来练习可以不用在笔记本上开多个虚拟机 / Vargnet / Docker 服务，创建和删除都很方便，付费只需要按照用量付费，和网吧上网似的，多合适。
